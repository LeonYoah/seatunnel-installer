# UI 改造任务说明

## 目标

将当前 SeaTunnel Installer Web 控制台改造成类似 Doris 控制台的统一、干净的企业级监控 UI：

- 左侧窄导航 + 顶部二级标签导航
- 大量留白、轻边框/阴影的卡片化布局
- 统一的颜色/间距/字号/状态样式
- 关键页面：总览（Dashboard）、主机（Hosts）等

---

## 阶段 1：设计规范与基础样式

- [ ] **设计 Token 整理**
  - [ ] 在 `web/src/styles` 中新增/梳理 Token 文件（例如 `_tokens.scss`）
  - [ ] 定义统一的颜色变量：主色、成功/警告/错误、中性色等
  - [ ] 定义统一的圆角、阴影、间距、字体层级
- [ ] **覆盖主题与基础布局**
  - [ ] 在 `theme.scss` 中引用 Token，覆盖 Element Plus 默认主色/背景/边框
  - [ ] 调整 `MainLayout`、`Header`、`Sidebar` 的背景、边线、阴影，保证与目标 UI 一致
  - [ ] 统一内容区左右内边距（例如 24px），卡片间距使用 8/12/16/24px 阶梯

---

## 阶段 2：公共 UI 组件搭建

- [ ] **PageTabs（二级导航）**
  - [ ] 创建 `components/layout/PageTabs.vue`
  - [ ] Props：`tabs`（key/label/route）、`activeKey`（可从路由推导）、右侧操作 `#actions` 插槽
  - [ ] 样式：扁平标签 + 当前激活下边线，右侧为操作按钮区域
  - [ ] 在 Dashboard / Hosts 等页面顶部接入 `PageTabs`

- [ ] **StatCard（统计卡片）**
  - [ ] 创建 `components/common/StatCard.vue`
  - [ ] 支持：图标、数值、标签、状态（决定颜色），必要时拓展副标签
  - [ ] 用 `StatCard` 替换 `Dashboard.vue` 中 4 个 KPI 卡片的重复结构

- [ ] **InfoCard（信息展示卡）**
  - [ ] 创建 `components/common/InfoCard.vue`
  - [ ] 模式：标题 + `#default` 插槽，内部 label-value 布局
  - [ ] 在需要展示「连接信息」「集群信息」等块时统一使用该组件

---

## 阶段 3：总览页（Dashboard）改造

- [ ] **结构与布局**
  - [ ] 顶部增加 `PageTabs`，用于：总览 / 节点 / 主机 / 任务等
  - [ ] 重构 KPI 区域为 3–4 列栅格，保持统一高度与间距
  - [ ] 图表与告警卡片使用统一的 `el-card` 与 InfoCard 风格

- [ ] **视觉细节**
  - [ ] 使用 Token 控制卡片圆角/阴影，弱化 Element 默认描边
  - [ ] 统一数字字号（大号）、标签字号（中号）、说明文字（小号）
  - [ ] 告警列表使用统一的状态 Tag 颜色（WARN/ERROR 等）

---

## 阶段 4：主机页（Hosts）改造

- [ ] **整体框架**
  - [ ] 在 `Hosts.vue` 顶部接入 `PageTabs`，右上放「刷新」「集群伸缩」「重启」等操作（可先占位）
  - [ ] 将当前大 `el-card` 内部布局调整为：工具栏 + 表格 + 分页

- [ ] **表格与详情结构**
  - [ ] 使用 `el-table-column type="expand"` 实现行内展开详情
  - [ ] 在展开区域中展示：CPU 型号、内存总量、网卡信息、注册时间等
  - [ ] 在展开区域中增加磁盘子表格：磁盘名 / 剩余空间 / 总容量 / 利用率进度条 / IO Util
  - [ ] 主行维持：IP、Agent 状态、主机状态、CPU/内存利用率、网络 IO、Average Load 等

- [ ] **视觉与交互**
  - [ ] 统一表头背景和边线，提升可读性
  - [ ] CPU/内存进度条使用统一颜色规则：正常（绿）/偏高（橙）/告警（红）
  - [ ] 预留高负载行的特别高亮（例如浅红背景或左侧色条）

---

## 阶段 5：其他页面与整体打磨

- [ ] **其他核心页面统一**
  - [ ] 将 `Tasks.vue`、`Diagnostics.vue`、`Clusters.vue` 等页面接入 `PageTabs` 和统一卡片/表格风格
  - [ ] 梳理所有页面的状态色使用，统一为：运行中/正常/告警/失败/暂停 等标准枚举

- [ ] **体验优化**
  - [ ] 考虑支持「紧凑模式」表格行高切换（适合主机很多的场景）
  - [ ] 增加国际化文案与 UI 一致性检查（字段名、列名、提示信息）
  - [ ] 补齐关键交互提示（刷新/重启/伸缩等）和 Loading 状态

---

## 备注

- 本文档只描述 UI/交互层改造，不覆盖后端接口与数据对接。
- 实施过程中，可按页面拆分 PR：先基础样式与组件，再逐页替换，降低风险。
